# Component Patterns

Rules for building registry components in `registry/new-york/ui/`.

## File Structure

Every component is a folder with exactly 2 files:

```
registry/new-york/ui/{name}/
├── {name}.tsx    # Structure + variants (CVA)
└── {name}.css    # Theming (colors, borders, shadows)
```

Optional third file for documentation:
```
├── doc.md        # Component usage guide (generated by /component doc)
```

## TSX Template

```tsx
import * as React from "react"
import { cva, type VariantProps } from "class-variance-authority"

import { cn } from "@/lib/utils"
import "./{name}.css"

const componentVariants = cva("component-base", {
  variants: {
    variant: {
      primary: "component-primary",
      secondary: "component-secondary",
    },
    size: {
      sm: "component-sm",
      md: "component-md",
    },
  },
  defaultVariants: {
    variant: "primary",
    size: "md",
  },
})

function Component({
  className,
  variant,
  size,
  ...props
}: React.ComponentProps<"element"> &
  VariantProps<typeof componentVariants>) {
  return (
    <element
      data-slot="component"
      className={cn(componentVariants({ variant, size, className }))}
      {...props}
    />
  )
}

export { Component, componentVariants }
```

## Import Order

Strict order, no exceptions:

1. `import * as React from "react"` (always first)
2. Radix primitives (if needed): `import * as DialogPrimitive from "@radix-ui/react-dialog"`
3. Third-party: `import { cva, type VariantProps } from "class-variance-authority"`
4. Internal: `import { cn } from "@/lib/utils"`
5. CSS (always last): `import "./{name}.css"`

## data-slot Convention

Required on every component root element and every sub-component.

```tsx
// Root component
<div data-slot="tag" ... />

// Sub-components use kebab-case
<span data-slot="tag-dot" ... />
<button data-slot="tag-close" ... />
```

Rules:
- `React.ComponentProps<"element">` for native elements
- `VariantProps<typeof variants>` for CVA variants
- Custom props inline, not in a separate type (unless complex)
- Always destructure `className` and spread `...props`
- No `forwardRef` unless Radix or explicit ref forwarding needed

## Exports

Flat exports, never namespaced:

```tsx
// CORRECT
export { Tag, TagDot, TagClose, tagVariants }

// WRONG
export default Tag
export const Tag = { Root: ..., Dot: ..., Close: ... }
```

## CVA Base Class

The base string in `cva()` should contain:
- The CSS class name for shared styling (e.g., `"btn-base"`)
- Structural Tailwind classes (flex, overflow, cursor, transition, leading)
- NOT color/spacing classes — those go in the CSS file

```tsx
// CORRECT — structure in TSX, theming in CSS
const buttonVariants = cva(
  "btn-base inline-flex items-center justify-center cursor-pointer transition-colors",
  { ... }
)

// WRONG — theming in TSX
const buttonVariants = cva(
  "bg-blue-500 text-white rounded-lg p-4",
  { ... }
)
```

## Variant CSS Class Names

CVA variant values should be single CSS class names that delegate to the CSS file:

```tsx
variants: {
  variant: {
    brand: "tag-brand",     // styles defined in tag.css
    neutral: "tag-neutral",
    danger: "tag-danger",
  },
}
```

## Sub-Components

Keep sub-components minimal. CSS provides all styling.

```tsx
function TagDot({ className, ...props }: React.ComponentProps<"span">) {
  return <span data-slot="tag-dot" className={cn("tag-dot", className)} {...props} />
}
```

## Compound Components (Radix-based)

For components wrapping Radix primitives (Modal, Select, Tooltip):

```tsx
import * as DialogPrimitive from "@radix-ui/react-dialog"

// Root re-export
const Modal = DialogPrimitive.Root
Modal.displayName = "Modal"

// Enhanced sub-component
const ModalContent = React.forwardRef<...>(({ ... }, ref) => (
  <DialogPrimitive.Portal>
    <DialogPrimitive.Overlay className="modal-overlay" />
    <DialogPrimitive.Content
      ref={ref}
      data-slot="modal-content"
      className={cn("modal-content", sizeClass, className)}
      {...props}
    >
      {children}
    </DialogPrimitive.Content>
  </DialogPrimitive.Portal>
))
```

## Spacing in TSX

Inside registry components, always use token-based Tailwind arbitrary values:

```tsx
// CORRECT — token-based
className="gap-[var(--layout-gap-sm)] px-[var(--layout-padding-md)]"

// WRONG — raw Tailwind spacing
className="gap-2 px-4"
```

Exception: Doc pages (`app/components/*/page.tsx`) can use raw Tailwind classes.
